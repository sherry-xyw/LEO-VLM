# 产业与任务案例：在轨持续适应

> **用途**：本文件记录可用于界定工程边界的在轨任务、飞行代表执行和星座系统案例。它们不替代算法论文，也不能单独证明某种 LoRA、RAG 或联邦聚合算法已经在轨适用。
>
> **关键结论**：截至当前已核验材料，已有 GeoFM 与零样本 VLM 的在轨推理证据，但尚未找到同时公开证明“在轨 RS-VLM 参数高效微调 → 多星聚合 → 任务级验证 → 版本回滚”的端到端飞行案例。因此，本章应把在线微调、星座协同与版本治理明确写成研究空白，而不是既有能力。

## 已核验任务与工程证据

| 案例 | 直接支持的事实 | 与持续适应的关系 | 不可外推的内容 | 证据等级 |
| --- | --- | --- | --- | --- |
| **NAVI-Orbital**（2026） | 2026 年 4 月 16 日，在低轨航天器上完成本地 Gemma 3 零样本多模态推理：场景分类、文本描述和自然语言交互；系统可通过自然语言提示重新设定任务 | 证明 VLM 的在轨本地执行、任务重设和状态机编排可行；可作为 Prompt/任务策略热更新的系统边界 | 未微调飞行仪器；未比较 LoRA/Adapter、RAG 增量、训练能耗、星座聚合或回滚 | A：已在轨推理 |
| **IMAGIN-e / 压缩 GeoFM**（2025） | 压缩 GeoFM 在 ISS IMAGIN-e 载荷完成可靠在轨推理；论文同时强调压缩与跨传感器域适配的重要性 | 证明“飞行前压缩 + 域适配 + 在轨执行”是可行链路；提示持续适应要面对传感器域差异 | 不是在线学习或在轨微调；没有多星聚合和记忆合并 | A：已在轨推理 |
| **LoRA-Det**（2024） | 星载有向目标检测设定中，仅更新约 12.4% 参数即可达到全量微调约 97%–100% 的性能 | 最接近“传输参数增量而非全模型”的单星 PEFT 证据 | 非生成式 VLM；未报告真实在轨训练、链路窗口或版本回滚 | D：地面遥感实验 |
| **FEDMEGA**（2024） | LEO 星座内采用轨道内 ring all-reduce 与全局流量调度；仿真报告相对对比方法约 30% 收敛速率改善 | 可作为“轨道内先聚合 adapter/LoRA delta，再做星地协调”的网络骨架 | 不是 VLM、LoRA 或真实星座部署 | D：LEO 系统仿真 |
| **AsyncFLEO**（2022） | 利用卫星分组、模型传播和陈旧度折扣处理可见窗口错位；仿真中减少同步等待 | 定义异步 adapter 更新的 freshness/staleness 权重和分组逻辑 | 不是多模态模型、在轨实测或飞行级安全发布流程 | D：LEO 系统仿真 |
| **SFL-LEO**（2025） | 断连期间继续局部训练，后续在地面聚合 client-side 子模型；以 Starlink 带宽测量驱动仿真 | 当单星无法完成完整 PEFT 训练时，为 split-PEFT 提供候选退路 | 未建模高分辨率 VLM 激活传输、RAG 和记忆一致性 | D：LEO 系统仿真 |

## 对正文的正确使用方式

### 1. 证明“能运行”与证明“能持续适应”必须分开

NAVI-Orbital 和 IMAGIN-e 可以支撑“卫星级边缘平台已能执行基础模型推理”；但这不能推出模型已能够在轨持续更新。持续适应额外需要训练时内存、热稳态、标签/伪标签来源、更新包完整性、验证集、版本控制和回滚证据。

### 2. Prompt 重设不等于 Prompt tuning

NAVI-Orbital 的自然语言重设任务说明**任务策略可被运行时配置改变**；但这不是对 soft prompt、prefix 或视觉 prompt 的梯度更新。综述中应将两者分别归入“任务重配置”和“参数高效适配”。

### 3. 域适配不等于持续学习

IMAGIN-e 说明跨传感器域差异会影响飞行环境中的基础模型表现，因此域适配是必要的；但其适配主要发生在飞行前。它可以说明持续适应的需求来源，不能作为在线训练成果。

### 4. 星座论文应作为通信与调度证据

FEDMEGA、AsyncFLEO 和 SFL-LEO 支撑轨道内聚合、异步接触计划、陈旧更新折扣和模型切分的系统设计；它们不能替代联邦 LoRA、联邦 RAG 或 RS-VLM 遗忘控制的算法证据。

## 面向飞行级持续适应的最小验证闭环

后续任何声称“可在轨持续适应”的系统，至少应报告：

```text
变化事件/漂移数据
  → 本地检测与更新触发
  → RAG/Prompt/Adapter/LoRA 更新
  → 新旧任务联合验证
  → 更新包签名、版本号与完整性校验
  → 异步上传或轨内聚合
  → 灰度下发、影子推理与回滚
```

| 环节 | 最低可核验指标 |
| --- | --- |
| 触发 | 检测延迟、误报率、漏报率、触发后收益/成本比 |
| 本地更新 | 可训练参数比例、训练时长、峰值内存、能耗、热状态 |
| 通信 | adapter/索引差分字节数、窗口完成率、staleness 分布 |
| 验证 | 新任务收益、旧任务 forgetting、置信度、grounded answer accuracy |
| 发布 | 版本一致率、A/B 通过率、回滚时间、失败恢复率 |

## 原始来源与引用建议

- **NAVI-Orbital**：Delfa Victoria et al., *NAVI-Orbital: First In-Orbit Demonstration of a Zero-Shot Vision-Language Model for Autonomous Earth Observation*, arXiv:2606.18271, 2026。建议仅用于“在轨 VLM 推理与任务重设”证据。
- **IMAGIN-e**：Du et al., *First On-Orbit Demonstration of a Geospatial Foundation Model*, arXiv:2512.01181, 2025。建议仅用于“压缩与域适配后在轨执行”证据。
- **LoRA-Det**：Pu and Xu, arXiv:2406.02385, 2024。建议用于“单星参数增量更新”的遥感任务证据。
- **FEDMEGA**：Shi et al., arXiv:2404.01875, 2024。建议用于“轨道内聚合”的系统证据。
- **AsyncFLEO**：Elmahallawy and Luo, arXiv:2212.11522, 2022。建议用于“异步聚合与陈旧度控制”的系统证据。
- **SFL-LEO**：Wu et al., arXiv:2504.13479, 2025。建议用于“算力不足时的切分训练”边界。

## 使用边界

- 产品规格、新闻稿和厂商案例只作为 E 级趋势线索；
- 任务方技术报告、在轨演示论文或官方工程文档可作为 B/A 级工程证据，但需注明任务日期、实际载荷与指标口径；
- 不以“支持 OTA / 在轨重配置”推导“支持在轨 VLM 微调”；
- 不以“已在轨推理”推导“已在轨持续学习”；
- 不以星座联邦仿真替代真实 VLM 训练的通信、能耗和热控结果。

> **目录定位**：本目录与 `单星部署-轻量化VLM与硬件感知/` 平行，服务于综述中“持续适应：在轨微调”部分。重点不是泛泛讨论持续学习，而是研究视觉—语言模型如何在低轨单星与星座环境中，以受控资源、受限链路和可回滚机制持续吸收新观测、新任务和新知识。

## 章节边界

### 本章重点

- 任务、模态、地域与季节变化下的分布漂移检测及更新触发；
- RAG、记忆库、知识图谱或任务缓存的非参数化更新；
- Prompt、Adapter、LoRA 等参数高效在线适配；
- 单星增量学习中的遗忘控制、样本选择与安全验证；
- 星座条件下的联邦 LoRA、异步聚合、参数增量压缩与多星记忆合并；
- 版本管理、A/B 验证、回滚、异常检测和更新后可靠运行。

### 不在本章展开

- 模型量化、KV cache、分块推理、early exit 等推理期轻量化：归入“单星部署”；
- 星间模型切分与跨星激活传输：归入“模型切分与协同推理”；
- 一次性离线任务微调：仅作为对照基线，不作为持续适应主体。

## 核心问题

```text
观测、任务或知识发生变化
  → 漂移识别与更新触发
  → 选择更新对象：检索库、Prompt、Adapter/LoRA 或模型子模块
  → 单星局部验证与安全门控
  → 星座增量同步、异步聚合或记忆合并
  → 版本发布、监测与可回滚运行
```

## 当前调研结论

1. 在轨持续适应应构造成分级闭环：RAG/记忆更新优先，Prompt/Adapter/LoRA 局部适配随后，星座协同聚合与安全发布提供一致性保障。
2. RS-VLM 已有专门持续学习基准；但现有公开证据仍缺少同时覆盖在轨训练、星座聚合、版本回滚与任务级长期验证的完整飞行级系统。
3. 联邦 LoRA 不宜直接平均低秩因子；异构 rank、模态缺失、非 IID 数据与陈旧更新需要采用子空间/维度级聚合与个性化 readout。
4. 详细证据、适用边界和正文可用结论见 [`持续适应-核心文献.md`](持续适应-核心文献.md)。

## 二、技术分类与 Ch.6 对应关系

| Ch.6 子问题 | 核心论文 | 可提取论点 | 星载证据强度 |
| --- | --- | --- | --- |
| 6.1 变化建模与更新触发 | CLeaRS、CoSMAE；补充 E4 | 需区分长时程、模态、任务、知识和资源变化；漂移类型决定保留还是遗忘历史样本 | 弱：RS 地面基准和通用 FL |
| 6.2 非参数知识更新 | RS-RAG、SAR-RAG；补充 E2 | 新知识、案例与区域语义先进入向量库/摘要/证据 Prompt，再判断是否改参数 | 弱：地面 RS-VLM |
| 6.3 单星 PEFT | LoRA-Det、GeoFM PEFT、SpectralGPT | 低秩、Kronecker 或局部模块更新能显著缩小参数增量；应按视觉、连接器、语言和任务头分部件预算 | 中：任务面向星载，但未有在轨训练闭环 |
| 6.4 遗忘控制与局部验证 | CLeaRS、CoSMAE；补充 E3、E5 | PEFT 不是遗忘控制本身；需回放/蒸馏/正交或隔离更新，并做新旧任务联合验证 | 弱：地面 RS 或通用模型 |
| 6.5 卫星—星地协同微调 | Satellite FTF、On-board FL、FEDMEGA、AsyncFLEO、SFL-LEO、Sparse IA | 联邦/切分训练必须服从接触窗口、拓扑、链路和计算拆分，不能假设同步 FedAvg | 中：卫星网络架构/仿真，非飞行实证 |
| 6.6 异构 LoRA 与记忆合并 | FediLoRA、ILoRA、GLoRA；补充 E2、E5 | 不同 rank、模态和任务不能朴素平均 A/B 因子；应聚合可对齐的子空间或选择性更新 | 弱：通用联邦实验 |
| 6.7 版本治理与发布 | 补充 E1、E2；工程要求 | 检索库、Prompt、Adapter、基座与聚合状态须独立版本化、验签、灰度和回滚 | 弱：尚无端到端在轨公开验证 |

## 1. Weng et al.：CLeaRS

- **出处**：Xingxing Weng et al. *Continual Vision-Language Learning for Remote Sensing: Benchmarking and Analysis*, arXiv:2604.00820, 2026。[原文](https://arxiv.org/abs/2604.00820)
- **任务**：系统评估 RS-VLM 在连续到达的时间、模态与任务变化中的适应和遗忘。
- **基础模型 / 硬件**：比较多种 RS-VLM；未给出星载训练硬件、功耗或热设计。
- **训练 / 更新方式**：将代表性持续学习方法适配到 VLM；关键是比较协议而非提出在轨训练器。
- **数据与场景**：10 个整理子集、20.7 万+图文对，覆盖不同遥感解释任务、传感器模态和应用场景。
- **输入 / 输出**：连续图像—文本/任务流输入；输出各阶段任务表现与遗忘统计。
- **Pipeline**：任务/模态流 → VLM 持续更新 → long-horizon、modality-incremental、task-incremental 三种评测 → 新旧能力分析。
- **主实验 / 指标**：所有评测设置都出现明显灾难性遗忘；既有方法对任务、指令和模态转换的改善有限。
- **硬件证据**：无。不能由该基准推导出在轨训练内存、能耗或窗口可行性。
- **局限**：是地面 RS-VLM benchmark；时间顺序和数据协议仍不能直接等价于具体卫星轨道、云量或能源状态。
- **对本综述用途**：Ch.6 的问题定义与最低评测基线；避免把“类别增量”误当作全部在轨变化。

## 2. Möllenbrok et al.：CoSMAE

- **出处**：Lars Möllenbrok, Behnood Rasti, Begüm Demir. *Continual Self-Supervised Learning with Masked Autoencoders in Remote Sensing*, arXiv:2506.21312, 2025。[原文](https://arxiv.org/abs/2506.21312)
- **任务**：在缺少持续人工标签的条件下，使遥感 MAE 学习新观测流而减少遗忘。
- **基础模型 / 硬件**：MAE 视觉表征模型；未涉及语言模块或星载硬件。
- **训练 / 更新方式**：data mixup 保留历史分布联系；model mixup knowledge distillation 通过混合历史/当前模型权重形成教师约束。
- **数据与场景**：连续遥感任务流上的无监督表征学习。
- **输入 / 输出**：当前与历史遥感图像输入；输出更新后的视觉表征与下游任务性能。
- **Pipeline**：无标注观测流 → 当前/历史数据混合 → MAE 更新 + 混合教师蒸馏 → 下游评测。
- **主实验 / 指标**：论文报告相对既有 MAE 持续学习方法最高约 4.94% 的提升。
- **硬件证据**：无训练内存、功耗、热或链路实验。
- **局限**：更新对象是视觉 MAE，而非生成式 VLM；数据混合和教师模型本身也占用星上存储与训练资源。
- **对本综述用途**：支撑“低可信或无标签观测先做视觉侧预适应，可靠监督到位后再触发任务级 PEFT”的两阶段策略。

## 3. Pu and Xu：LoRA-Det

- **出处**：Xinyang Pu, Feng Xu. *Low-Rank Adaption on Transformer-based Oriented Object Detector for Satellite Onboard Processing of Remote Sensing Images*, arXiv:2406.02385, 2024。[原文](https://arxiv.org/abs/2406.02385)
- **任务**：在上行带宽受限时，为卫星遥感有向目标检测器更新新数据分布。
- **基础模型 / 硬件**：Transformer-based oriented detector；论文讨论卫星在板处理需求，但不报告飞行平台训练实测。
- **训练 / 更新方式**：冻结原权重，训练低秩矩阵；并以低秩近似选择 LoRA rank。
- **数据与场景**：遥感有向目标检测及分布适配。
- **输入 / 输出**：遥感图像输入；输出旋转目标框和类别；传输/保存对象为低秩参数增量。
- **Pipeline**：地面或本地标注/新分布数据 → LoRA 更新 → rank 选择 → 下发/合并低秩权重 → 目标检测推理。
- **主实验 / 指标**：仅更新约 12.4% 的总参数，即达到全量微调约 97%–100% 的性能。
- **硬件证据**：带宽动机明确，但没有在轨训练时间、峰值内存、能耗、热状态或版本回滚实验。
- **局限**：判别式检测器不等同于包含视觉编码器、连接器和语言解码器的 RS-VLM；参数比例也不能直接外推为 VLM 更新包大小。
- **对本综述用途**：Ch.6 最直接的“星载任务优先传输参数增量而非全模型”证据，但必须补充资源与安全发布门控。

## 4. Marti-Escofet et al.：Geospatial Foundation Model PEFT

- **出处**：Francesc Marti-Escofet et al. *Fine-tune Smarter, Not Harder: Parameter-Efficient Fine-Tuning for Geospatial Foundation Models*, arXiv:2504.17397, 2025。[原文](https://arxiv.org/abs/2504.17397)
- **任务**：比较不同地理空间基础模型架构和 PEFT 方法在 EO 任务上的性能、训练资源与空间泛化。
- **基础模型 / 硬件**：多类地理空间基础模型与解码器；地面训练实验。
- **训练 / 更新方式**：比较多种 PEFT 与全量微调，同时考察解码器和元数据使用选择。
- **数据与场景**：5 个 EO 数据集，并检验未见地理区域的泛化。
- **输入 / 输出**：EO 图像及可选元数据输入；输出下游地理空间任务预测。
- **Pipeline**：冻结/部分冻结 GeoFM → 选择 PEFT 与解码器 → 任务训练 → 已见/未见区域评测 → 资源比较。
- **主实验 / 指标**：PEFT 可匹配或超过全量微调，降低训练时间与内存需求，并改善未见区域泛化；作者建议优先考察 UNet 解码器和不使用元数据的配置，但该结论受其模型/任务设定约束。
- **硬件证据**：没有星载平台、功耗、热与通信窗口测试。
- **局限**：是 EO 视觉基础模型适配，不涵盖 VLM 生成、在轨训练或星间协同。
- **对本综述用途**：支持“冻结基座 + 小模块适配”不仅节省通信，也可降低全量微调损害已有表征和空间泛化的风险。

## 5. Ligan et al.：SpectralGPT PEFT

- **出处**：Bernardin Ligan, Khalide Jbilou, Fahd Kalloubi, Ahmed Ratnani. *Parameter-Efficient Fine-Tuning of Multispectral Foundation Models for Hyperspectral Image Classification*, arXiv:2505.15334, 2025。[原文](https://arxiv.org/abs/2505.15334)
- **任务**：将多光谱基础模型 SpectralGPT 参数高效地适配到高光谱分类。
- **基础模型 / 硬件**：SpectralGPT 多光谱基础模型；地面高光谱分类实验。
- **训练 / 更新方式**：比较 LoRA、LoRA+、KronA、LoKr，并提出以差异学习率训练 Kronecker 因子的 KronA+。
- **数据与场景**：5 个来自不同传感器的高光谱数据集。
- **输入 / 输出**：高光谱影像输入；输出像素/样本分类；模型增量为 PEFT 模块。
- **Pipeline**：多光谱预训练表征 → 高光谱输入适配 → PEFT/全量微调对比 → 跨传感器分类评测。
- **主实验 / 指标**：KronA+ 在相同训练轮次下以约 0.056% 可训练参数、约 0.2 MB 增量存储取得接近全量微调的表现。
- **硬件证据**：未报告在板训练、热/能耗或在轨链路实验。
- **局限**：高光谱分类并非 VLM；极小增量不包含优化器状态、激活、回放缓存与验证数据的资源代价。
- **对本综述用途**：支撑新谱段、新载荷或跨传感器变化时，为视觉/模态模块配置比统一 LoRA 更小的更新预算。

## 6. Wen et al.：RS-RAG

- **出处**：Congcong Wen et al. *RS-RAG: Bridging Remote Sensing Imagery and Comprehensive Knowledge with a Multi-Modal Dataset and Retrieval-Augmented Generation Model*, arXiv:2504.04988, 2025。[原文](https://arxiv.org/abs/2504.04988)
- **任务**：让 RS-VLM 使用外部世界知识和领域知识，减少仅依赖封闭集场景理解的局限。
- **基础模型 / 硬件**：多模态遥感检索与 VLM 生成框架；无星载硬件报告。
- **训练 / 更新方式**：构建图文统一向量库，在推理时检索、重排序并将证据写入 knowledge-augmented prompt。
- **数据与场景**：RSWK 数据集包含 175 个国家的 14,141 个著名地标及高分辨率卫星图像和详细文本说明。
- **输入 / 输出**：图像和/或文本查询输入；输出描述、分类或 VQA 回答及其检索证据。
- **Pipeline**：图文知识编码 → 向量库 → 图像/文本查询 → 检索与重排 → 证据增强 Prompt → VLM 回答。
- **主实验 / 指标**：在图像描述、图像分类和 VQA 上优于论文所比较的基线。
- **硬件证据**：无星上索引增量、检索时延、向量库存储、冲突处理或离线更新验证。
- **局限**：检索质量不是参数能力的替代指标；知识过期、相邻区域混淆和证据冲突仍可能导致错误输出。
- **对本综述用途**：形成“知识变化优先更新向量库、摘要和证据 Prompt；仅在检索无法恢复能力时升级到 PEFT”的低风险路径。

## 7. Ramirez et al.：SAR-RAG

- **出处**：David F. Ramirez et al. *SAR-RAG: ATR Visual Question Answering by Semantic Search, Retrieval, and MLLM Generation*, arXiv:2602.04712, 2026。[原文](https://arxiv.org/abs/2602.04712)
- **任务**：通过检索历史 SAR 目标案例提升 ATR 相关问答、分类和尺寸估计。
- **基础模型 / 硬件**：MLLM + 语义向量数据库；无星载硬件/在轨实验。
- **训练 / 更新方式**：对带已知属性的历史目标样本建立语义 embedding，以相似案例增强 MLLM 的视觉问答。
- **数据与场景**：SAR 自动目标识别，重点是外观相近车辆类别的区分。
- **输入 / 输出**：当前 SAR 图像与查询输入；输出检索到的案例、目标类别、解释和数值属性预测。
- **Pipeline**：已标注 SAR 案例 → 向量化记忆库 → 当前样本语义搜索 → 相似案例检索 → MLLM 生成 ATR 回答。
- **主实验 / 指标**：作者以检索指标、分类准确率与目标尺寸数值回归评估，报告加入记忆库后相对 MLLM 基线有改善。
- **硬件证据**：没有星上向量索引更新、内存、功耗、对抗性案例或发布回滚测试。
- **局限**：面向特定 SAR ATR；案例库的标签正确性、来源追溯和地域/时相失效在在轨应用中必须单独治理。
- **对本综述用途**：把“记忆”具体化为带来源和传感器元数据的目标案例，而非仅更新开放世界文本知识。

## 8. Zhu et al.：Satellite Federated Fine-Tuning

- **出处**：Yan Zhu et al. *Satellite Federated Fine-Tuning for Foundation Models in Space Computing Power Networks*, arXiv:2504.10403, 2025。[原文](https://arxiv.org/abs/2504.10403)
- **任务**：在单星算力不足、星地窗口短且 ISL 不稳定的情况下，支持基础模型的卫星—地面协同微调。
- **基础模型 / 硬件**：以遥感基础模型为对象；论文将冻结的嵌入层和可训练任务头放在卫星，将约占主要计算负担的冻结骨干放在地面。未报告真实在轨设备训练。
- **训练 / 更新方式**：卫星—地面协同的 **head-only** 联邦微调；卫星与地面交换嵌入/特征向量，轨内聚合任务头；提出并行轨内、拓扑感知星地和跨轨通信策略。
- **数据与场景**：空间算力网络与 LEO 通信拓扑仿真。
- **输入 / 输出**：卫星本地影像输入；星地交换嵌入/特征向量，卫星间交换并聚合任务头状态，而非只交换独立 LoRA delta。
- **Pipeline**：影像 → 卫星嵌入层 → 轨内嵌入向量广播/星地传输 → 地面冻结骨干提取特征 → 特征回传 → 卫星任务头更新 → 轨内/跨轨 head 聚合。

- **局限**：该方案仅训练任务头，且星地传输的嵌入/特征向量可能超过 PEFT 参数增量；系统结论不能替代针对具体 VLM 层切分点的资源 profiling。
- **对本综述用途**：当单星无法完成完整 PEFT 时，作为“冻结骨干 + 卫星任务头”的拆分训练备选；需先比较 `embedding/feature bytes` 与 `adapter delta bytes`，而非默认采用切分训练。

```text
观测、任务、传感器、知识或资源状态变化
  → 本地漂移 / 不确定性 / 检索失效 / 资源预算检测
  → 变化归因：数据、模态、任务、知识或系统资源
  → 选择最小更新单位：记忆/RAG → Prompt → Adapter/LoRA/连接器/任务头
  → 单星局部训练或自监督预适应（资源门控）
  → 新旧任务联合验证、证据一致性检查、包完整性校验
  → 生成带版本的 adapter delta、索引差分或案例摘要
  → 按载荷/区域/季节/任务/rank 分组的轨内异步协同
  → 子空间/维度级选择性聚合、星地全局复核
  → 签名、影子推理、灰度发布、稳定版本保留和可回滚下发
  → 持续监测版本年龄、性能退化、资源余量和新漂移
```

### 更新策略原则

1. **先判断变化，后决定是否更新**：性能波动可能来自云雾、观测几何、检索失效、任务规则或资源降级；它们不都需要训练。
2. **知识变化优先进入记忆层**：区域规则、事件报告、目标案例和载荷元数据变化时，先更新带来源的索引、摘要和证据 Prompt。
3. **短期任务变化优先采用 Prompt/策略文件**：输出格式、告警阈值和临时任务说明变化时，避免不必要的权重更新。
4. **持续域失配或新模态再触发 PEFT**：仅当低风险层不足以恢复能力、监督可信且收益超过训练/验证/通信代价时，更新 LoRA、Adapter、连接器或任务头。
5. **局部验证先于星座共享**：未通过新旧任务、资源、版本和完整性检查的候选更新不得进入轨内聚合。
6. **异构性分组而非一刀切平均**：按载荷、地区、季节、任务、可用资源和 LoRA rank 形成协同组；聚合语义可对齐的更新子空间或选择性 adapter。
7. **所有可变对象均须可回滚**：基座、Prompt、Adapter/LoRA、索引、案例摘要、聚合状态和策略文件独立编号、签名、灰度和保留稳定版本。

## 五、验证与报告规范

| 维度 | 最低报告项 | 常见缺口 |
| --- | --- | --- |
| 变化与触发 | 漂移类型、检测信号、检测延迟、误报/漏报、阈值、触发收益/成本比 | 只报告最终精度，不报告何时和为何更新 |
| 更新对象 | RAG、Prompt、Adapter、LoRA、连接器、任务头；可训练比例、包大小和合并方式 | 将“参数比例”误当作完整训练成本 |
| 数据可信度 | 标签来源、伪标签阈值、回放样本策略、案例来源/过期/冲突标记 | 默认线上观测都可训练或可写入记忆 |
| 单星资源 | 峰值训练内存、优化器/激活/回放占用、训练/验证时长、能耗、热状态、窗口完成率 | 只测推理或只测参数存储 |
| 新旧能力 | 新任务与旧任务指标、平均准确率、forgetting、forward/backward transfer、置信度校准 | 仅报告新分布收益 |
| 检索可信度 | Recall@k、证据一致性、grounded answer accuracy、冲突率、索引刷新/检索时延 | 将“有检索”视为事实可靠 |
| 星座协同 | 星间/星地字节数、激活/梯度与 delta 的分别统计、staleness 分布、版本分叉、群组公平性、to-target-accuracy 时延 | 把卫星拓扑简化为同步 FedAvg |
| 发布安全 | 影子评测、灰度比例、验签/完整性、版本一致率、回滚时间、失败恢复率 | 把一次离线微调当作可部署更新 |

### 必要对比基线

1. 静态冻结模型；
2. 仅 RAG/案例记忆更新；
3. 仅 Prompt/策略文件更新；
4. 单星 Adapter/LoRA/连接器更新；
5. 单星 PEFT + 回放/蒸馏/隔离的遗忘控制；
6. 本地完整 PEFT 与 split-PEFT，并分别报告激活/梯度通信；
7. 同步、异步和轨内增量聚合；
8. 朴素 LoRA 因子平均与维度级/子空间/选择性聚合；
9. 直接发布与影子评测、灰度和回滚发布。

## 六、可直接写入 Ch.6 的结论

1. 在轨持续适应的最小更新单位不应是完整模型，而应按变化来源选择知识索引、案例记忆、Prompt、Adapter、LoRA、连接器或局部任务头。
2. 遥感 VLM 的连续变化同时包括时间/地域观测分布、传感器模态、任务指令、外部知识和系统资源；单一类别增量协议不足以代表在轨环境。
3. RAG 与案例记忆是低风险的优先更新层，但它们不能替代对持续域失配和跨模态对齐问题的参数适配；检索失效、冲突和过期是从记忆层升级到 PEFT 的明确门槛。
4. PEFT 只压缩可训练权重，并不自动解决优化器状态、激活、回放缓存、能耗、热状态、链路窗口和灾难性遗忘；飞行可用性必须以完整资源闭环验证。
5. 星座协同的关键不是减少模型字节数，而是使模型拆分、接触计划、陈旧度、时空 non-IID 数据、不同 rank、模态异构和更新语义共同进入聚合与调度决策。
6. 直接平均 LoRA `A/B` 因子既会产生异构 rank 问题，也可能受低秩分解坐标的不唯一性影响；更合理的研究方向是分组化、维度级或子空间级选择性聚合，并保留个性化 readout。
7. 当前证据仍主要来自地面遥感实验与卫星网络仿真。将漂移触发、在轨 PEFT、记忆治理、异步协同、验证、验签、灰度和回滚整合为可长期运行的飞行级闭环，仍是尚未解决的研究问题。

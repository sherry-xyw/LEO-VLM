# 单星部署：轻量化 VLM 与硬件感知

> **文档定位**：本文件给出可直接写入综述正文的章节结构与写作组织方式。它不按论文逐篇汇报，也不把量化、剪枝、token 压缩等方法并列罗列；而是围绕“资源受限单星如何形成可运行、可验证、可恢复的 VLM 推理闭环”展开。现有 `单星部署.md` 中的文献细节、方法介绍与扩展表格可作为本章的证据库。

## 章节主旨

低轨单星上的 VLM 部署不是将地面模型直接压缩后迁移到卫星，而是一个由**任务输入、模型结构、序列长度、平台资源和飞行可靠性**共同决定的协同设计问题。正文应突出如下论点：

> **单星 VLM 的核心不是单点模型压缩，而是围绕高分辨率视觉输入、峰值内存、功率/热约束与故障恢复，对感知—推理—运行时—输出全链路进行硬件感知重构。**

建议篇幅：约 5–7 个综述正文页；其中方法机制约占 60%，硬件协同与评估约占 30%，小结与研究挑战约占 10%。

## 1. 问题定义：资源受限单星中的 VLM 推理闭环

### 1.1 从“模型上星”到“任务闭环”

**写作目标**：先定义本章讨论的对象，而不是直接进入压缩算法。

建议正文逻辑：

1. 星上任务的输入通常是高分辨率、多尺度、多光谱、多时相或视频流数据；
2. 对于 VLM，视觉编码器、跨模态连接器和语言解码器分别带来视觉 token、权重与 KV cache 三类负担；
3. 单星部署的最终目标不是离线吞吐最高，而是在规定任务窗口中稳定完成“输入处理—理解/生成—输出/下传—异常恢复”；
4. 因而部署结论必须同时闭合模型质量、时延、峰值内存、能量、热控与可靠性。

本节可使用一个统一的约束表达式作为全文总纲：

```text
M_peak ≤ M_available,
E_task ≤ E_available,
T_end-to-end ≤ T_deadline,
R_mission ≥ R_required.
```

其中，`M_peak` 是推理峰值内存，`E_task` 是任务能量消耗，`T_end-to-end` 是成像到输出的端到端时延，`R_mission` 表示故障条件下的任务连续性或可用率。

**引用组织**：用 Sang 的星上 RSFM 资源匹配框架说明“模型需求—平台资源—部署结论”的基本逻辑；用 VLM 推理文献支撑视觉 token 和 KV cache 是区别于传统 CNN 的关键资源负担。[R1, R4, R5]

### 1.2 单星部署的四类瓶颈

将后文所有技术统一映射到四类瓶颈，而不是按算法名称切分：

| 瓶颈 | 典型来源 | 对 VLM 的直接后果 | 后续对应技术 |
| --- | --- | --- | --- |
| 视觉工作负载 | 大幅面图像、多尺度目标、多时相输入 | token 数、视觉预填充计算和输入缓存快速增长 | ROI 筛选、分块、token 合并/剪枝、流式编码 |
| 峰值内存 | 权重、激活、视觉 token、KV cache、运行时工作区 | 模型可加载但无法稳定推理 | 量化、分层加载、KV 压缩、分页管理 |
| 持续功率与热控 | 有限单轨能量、散热能力、热降频 | 峰值性能无法转化为持续吞吐 | 动态分辨率、early exit、任务调度、功率模式管理 |
| 飞行可靠性 | TID/SEU/SEE、软件异常、存储损坏、链路中断 | 长周期任务中断或错误输出 | 校验、检查点、回滚、冗余与 CPU 回退 |

**写作要求**：这一节应建立“问题—方法”映射关系，避免把每一种优化写成独立、互不相关的技巧。

## 2. 面向星上资源约束的 VLM 执行链重构

**本章核心节。** 以模型从输入到输出的执行路径组织技术，而非以“量化、剪枝、蒸馏”等名词组织。

建议配图：

```text
星上原始观测
  → 任务感知预处理与 ROI 路由
  → 轻量视觉编码与 token 控制
  → 跨模态连接与小型语言解码
  → KV/内存感知运行时
  → 结构化输出、下传与故障回退
```

### 2.1 输入自适应与视觉 token 控制

**核心问题**：高分辨率遥感输入如何在不丢失小目标、细长目标与空间上下文的前提下进入 VLM？

建议从三层展开：

1. **任务前置筛选**：云/模糊过滤、轻量检测器、显著区域或变化区域筛选；
2. **空间分解**：全局缩略图 + 局部高分辨率 ROI + 空间位置编码；
3. **token 选择与压缩**：patch merging、token merging、显著性/指令感知 pruning、窗口注意力和层级视觉编码。

本节的关键论点应是：

> 遥感 VLM 的 token 压缩不应追求均匀削减，而应实现“全局语境保留—局部高价值区域增强—冗余背景压缩”的非均匀资源分配。

**可引用的代表性证据**：ToMe 说明 token 合并可提升 ViT 吞吐；LLaVA-UHD 提供任意分辨率图像的模块化、压缩和空间组织方案。[R3, R4]

**本节宜报告的指标**：视觉 token 数、ROI 覆盖率、小目标召回、边界误差、prefill 时延、输入缓冲峰值。

### 2.2 视觉—语言模型结构的紧凑化

**核心问题**：如何在不破坏跨模态对齐和任务能力的条件下缩减模型权重与计算？

建议将技术按“模型部件”而非按论文分类：

| 模型部件 | 主要约束 | 优先轻量化策略 | 写作重点 |
| --- | --- | --- | --- |
| 视觉编码器 | 高分辨率计算与中间激活 | 混合精度、结构化剪枝、蒸馏、层级编码 | 保留空间细节和小目标特征 |
| 跨模态连接器 | 表征对齐与投影开销 | 窄连接器、低秩投影、混合精度 | 防止视觉—语言对齐退化 |
| 语言解码器 | 权重、带宽与解码计算 | W8/W4 量化、GQA/MQA、小型解码器或受限生成 | 兼顾生成质量与内存预算 |
| 任务输出头 | 固定字段或低复杂度输出 | 结构化头、模板化生成、任务专用蒸馏 | 用任务约束替代开放式冗长生成 |

这里的论证顺序应是：先解释模型不同部位的资源与任务敏感性不同，再给出分部件、混合精度、结构化压缩的必要性；不宜直接给出“所有层均 INT4”的统一方案。

**可引用的代表性证据**：AWQ 与 SmoothQuant 支持低比特权重/激活量化的基本思路；GQA 表明减少 KV 头数可在速度与质量之间形成中间折中。[R2, R10, R6]

### 2.3 生成阶段的 KV 与长序列运行时管理

**核心问题**：如何控制生成长度、上下文与 KV cache 对单星峰值内存的持续侵占？

建议按“源头减量—表示压缩—内存管理—上下文裁剪”组织：

1. **源头减量**：使用 GQA/MQA、限定输出格式与最大生成长度；
2. **表示压缩**：KV 量化、关键 token 保留、历史 token 降精度；
3. **运行时管理**：paged KV、预分配块、避免碎片和重复缓存；
4. **上下文策略**：系统指令、关键视觉 anchor、近期窗口与历史摘要分层保留。

**核心论点**：对于短输出的筛选任务，优先降低视觉 token；对于问答、解释和报告生成任务，KV 管理是能否稳定运行的必要条件。

**可引用的代表性证据**：KIVI 讨论低比特 KV 量化；PagedAttention 处理 KV cache 的碎片与分页；StreamingLLM 与 MInference 分别提供流式上下文和长序列 prefill 的运行时思路。[R5, R7, R8, R9]

**本节宜报告的指标**：KV 峰值、缓存命中/淘汰策略、prefill/decode 分离时延、token/J、输出质量与事实一致性。

## 3. 硬件感知编译、运行时与动态资源治理

### 3.1 从“理论压缩率”到“物理加速比”

**写作目标**：强调模型 FLOPs、参数量或稀疏率下降并不自动转化为星上时延和能耗下降。

应说明：

- 非结构化稀疏若不被硬件内核和编译器识别，可能仍以稠密形式执行；
- 低比特权重只有在目标平台支持相应整数矩阵乘、数据布局和 kernel fusion 时才可转化为真实收益；
- 同一模型在 GPU、VPU、FPGA/MPSoC 上的最佳量化位宽、分块大小和算子划分不同。

建议正文采用“**模型配置—编译表示—硬件执行**”三层表述，而不是单列 TensorRT、TVM、Vitis AI、OpenVINO 等工具名。

| 层级 | 要解决的问题 | 应报告的内容 |
| --- | --- | --- |
| 模型配置 | 位宽、剪枝结构、token 上限、GQA 配置 | W/A/KV 位宽、稀疏结构、最大 token 数 |
| 编译与图优化 | 算子融合、数据布局、内存复用、静态/动态形状 | 支持算子比例、回退算子、图优化策略 |
| 硬件执行 | 核利用率、带宽、温度、功耗模式 | 实测时延、能耗、温度、降频、吞吐 |

### 3.2 功率—热控约束下的动态执行

将 early exit、动态分辨率、动态 token 预算和模型回退放在同一小节，统一为**动态资源治理**。

推荐提出四级执行模式：

| 模式 | 任务形态 | 执行链 | 资源目标 |
| --- | --- | --- | --- |
| M0 | 数据质量筛选 | 规则/轻量 CNN/小型视觉头 | 最低能耗、最大覆盖 |
| M1 | 视觉判别 | 轻量视觉编码器 + 任务头 | 快速分类、定位或变化检测 |
| M2 | 受限多模态输出 | 压缩 VLM + 结构化短输出 | 告警、属性归纳、固定字段报告 |
| M3 | 高价值复杂理解 | 完整或较完整 VLM 推理 | 复杂问答、解释和证据汇总 |

运行时可根据电量、温度、可用 DRAM、任务时限、链路状态和模型置信度在 M0–M3 间切换。这样，early exit 不再只是网络结构技巧，而成为星上任务调度机制的一部分。

### 3.3 飞行可靠性与降级运行机制

本节只讨论**推理期可靠性**；LoRA、Adapter、漂移检测和在轨更新算法留给“持续适应：在轨微调”章节详细讨论。

建议包括：

- 模型与权重包的哈希校验；
- checkpoint 与恢复点；
- A/B 版本及安全回滚；
- 主模型异常时的轻量模型或 CPU 回退；
- 对关键输出附带置信度、执行模式和资源状态；
- 针对 SEU/SEE、内存错误、热降频和链路中断的故障注入测试。

**核心论点**：对长期在轨运行而言，可靠降级比在理想环境中的峰值准确率更接近真实部署能力。

## 4. 面向单星任务的部署配置与策略选择

**写作目标**：把前文方法收敛为可操作的部署组合，而不是再列新技术。

| 任务/输入条件 | 建议模型形态 | 优先优化组合 | 不宜采用的策略 |
| --- | --- | --- | --- |
| 高频数据质量筛选 | 轻量视觉模型或视觉编码器 + 小头 | INT8、固定输入、early exit、M0 | 长文本生成、复杂 KV 管理 |
| 大幅面遥感理解 | 分块视觉编码器 + ROI 路由 + 短输出 VLM | 全局缩略图、ROI、token 合并、prefill 分段 | 将全部原图 patch 平权送入 LLM |
| 高光谱/多时相判别 | 流式编码 + 任务专用连接器 | 谱段选择、窗口缓存、分层加载 | 一次性拼接全部时相/谱段 |
| 高价值事件解释 | 小型 VLM 或分层调用 VLM | W4/W8、GQA、KV 量化、结构化输出、M2/M3 | 无限制开放生成 |
| 资源/温度异常 | 轻量模型回退或延迟执行 | 动态 token 预算、降级模式、checkpoint | 维持满载长上下文推理 |

本节应强调：**部署策略由任务价值、输入复杂度和可用资源共同决定，而不是由模型参数量单独决定。**

## 5. 单星部署的评测框架与报告规范

### 5.1 任务质量、资源效率与飞行可用性的联合评测

建议提出“单星部署报告卡”，每个方法至少报告以下五类结果：

| 维度 | 最低报告项 |
| --- | --- |
| 任务与输入 | 传感器、分辨率、波段/时相数、任务时效目标、输出长度 |
| 模型配置 | 参数量、视觉 token 上限、上下文长度、输出长度、量化位宽、GQA/KV 配置 |
| 资源效率 | 峰值 DRAM、存储占用、prefill/decode 时延、吞吐、J/image 或 J/task |
| 任务质量 | 任务精度、小目标召回、定位/边界误差、生成正确性、幻觉率、置信度校准 |
| 飞行可用性 | 热稳态吞吐、降频曲线、错误恢复时间、回退成功率、任务可用率 |

本节的贡献不在于再定义一个抽象 benchmark，而是明确指出：只报告地面端精度或 FLOPs，无法证明方案可在轨部署。

### 5.2 推荐对比基线

建议至少包含以下四类基线：

1. 未压缩模型或统一 FP16/BF16 模型；
2. 仅权重量化模型；
3. 仅视觉 token 压缩模型；
4. 全链路协同方案（量化 + token 控制 + KV 管理 + 运行时策略）。

在同一硬件、同一输入分布和同一任务时限下比较，才能识别“理论压缩”与“真实任务收益”的差异。

## 6. 小结：从算法压缩到飞行级协同部署

本章结尾建议用一段而非列表收束：

> 单星部署的关键不在于寻找单一最优压缩算法，而在于以任务闭环为单位，将视觉 token 控制、模型分部件轻量化、KV/内存管理、硬件编译、动态功率—热控调度和可靠降级机制共同纳入部署设计。对资源受限卫星而言，真正有价值的方案应同时证明其任务质量、资源效率与长期运行可用性。

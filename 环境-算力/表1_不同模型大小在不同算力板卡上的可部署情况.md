# 表 1：不同模型大小在不同算力板卡上的可部署情况

## 1. 目标、参数与配置

### 1.1 部署目标

视觉—语言模型（Vision-Language Model, VLM）在给定板卡、量化配置和参数取值下，可在单板、无中央处理器（Central Processing Unit, CPU）或 NVMe（Non-Volatile Memory Express） 固态存储卸载、无多卡并行的情况下稳定完成一次推理任务。

### 1.2 参数与定义

所有内存统一使用 GiB（gibibyte）：

$$
1\ \mathrm{GiB}=2^{30}\ \mathrm{Byte}.
$$

部分名词解释：B 表示十进制十亿，即 $1\ \mathrm{B}=10^9$ parameters。Byte 表示字节，$1\ \mathrm{Byte}=8$ bit。W4/W8 分别表示 4-bit/8-bit 权重量化；BF16 表示 Brain Floating Point 16-bit；TFLOPs 表示每秒万亿次浮点运算；FLOPs 表示浮点运算次数；TTFT（time to first token）表示首个 token 的生成时延；GPU、NPU 和 FPGA 分别表示图形处理器、神经网络处理器和现场可编程门阵列。

| 参数                                   | 含义                                                               | 取值（以Qwen3-VL-4B为例）                                          |
| ------------------------------------ | ---------------------------------------------------------------- | ----------------------------------------------------------- |
| $P_{\mathrm{LLM}}$                   | 语言主干参数量                                                          | 约 $3.7$B                                                    |
| $P_{\mathrm{vis}}$                   | 视觉编码器参数量                                                         | —                                                           |
| $P_{\mathrm{proj}}$                  | 跨模态投影器参数量                                                        | —                                                           |
| $P_{\mathrm{vis}}+P_{\mathrm{proj}}$ | 视觉编码器与投影器参数量总和                                                   | 约 $0.74$B                                                   |
| $P_{\mathrm{total}}$                 | 完整 VLM 总参数量（用于核对）                                                | 约 $4.44$B                                                   |
| $L$                                  | 语言模型 Transformer 层数                                              | 36                                                          |
| $H$                                  | 语言模型 hidden size                                                 | 2560                                                        |
| $n_{\mathrm{q}}$                     | 每层 query heads 数                                                 | 32                                                          |
| $n_{\mathrm{kv}}$                    | 每层 KV heads 数                                                    | 8                                                           |
| $d_h$                                | 每个 attention head 的维度                                            | 128                                                         |
| $D_Q$                                | attention 中 query 的总投影维度，$D_Q=n_{\mathrm{q}}d_h$                 | $32\times128=4096$                                          |
| $B$                                  | batch size                                                       | 1                                                           |
| $N_{\mathrm{text}}$                  | 文本输入 token 数                                                     | 1024                                                        |
| $N_{\mathrm{vis}}$                   | 实际输入语言模型的视觉 token 数                                              | 按模型 processor 实际记录；本文取值：1024                                |
| $N_{\mathrm{out}}$                   | 最大生成 token 数                                                     | 1024                                                        |
| $b^{\mathrm{eff}}$                   | 模块的实际有效存储位宽（bit/parameter）                                       | 语言主干 W4 为 $4.25$ bit、W8 为 $8.5$ bit；视觉编码器与投影器为 BF16（16 bit） |
| $b_{\mathrm{LLM}}^{\mathrm{eff}}$    | 语言主干的实际有效存储位宽                                                    | W4：$4.25$ bit；W8：$8.5$ bit                                  |
| $b_{\mathrm{vis}}^{\mathrm{eff}}$    | 视觉编码器的实际有效存储位宽                                                   | BF16：16 bit                                                 |
| $b_{\mathrm{proj}}^{\mathrm{eff}}$   | 跨模态投影器的实际有效存储位宽                                                  | BF16：16 bit                                                 |
| $b_{\mathrm{KV}}$                    | Key–Value（KV）cache 每个元素的存储位宽（bit）                                | BF16：16 bit；8-bit 浮点或 8-bit 整数格式：8 bit                      |
| $s_{\mathrm{KV}}$                    | Key–Value（KV）cache 每元素的存储字节数，$s_{\mathrm{KV}}=b_{\mathrm{KV}}/8$ | BF16：2 Byte                                                 |
| $M_{\mathrm{vision,peak}}$           | 图像与视觉编码器峰值内存                                                     | 示例假设：$0.5$ GiB（应由实际 profile 替换）                             |
| $M_{\mathrm{workspace}}$             | 推理 kernel 工作区内存                                                  | 示例假设：$0.6$ GiB（应由实际 profile 替换）                             |
| $M_{\mathrm{runtime}}$               | 框架与内存池等运行时内存                                                     | 示例假设：$0.5$ GiB（应由实际 profile 替换）                             |
| $M_{\mathrm{usable}}$                | 板卡可实际分配给推理进程的内存                                                  | 示例板卡 A/B/C：$6.8$ / $13.6$ / $27.2$ GiB                      |
| $F_{\mathrm{vision}}$                | 单图视觉编码计算量                                                        | 示例假设：$0.05$ TFLOPs（应由视觉编码器 profile 替换）                      |

推理参数集合记为 $\mathcal{C}$，定义为：

$$
\mathcal{C}=\{B,N_{\mathrm{text}},N_{\mathrm{vis}},N_{\mathrm{out}}\}.
$$

总输入长度 $T_{\mathrm{in}}$ 表示进入语言模型的文本 token 与视觉 token 总数，定义为：

$$
T_{\mathrm{in}}=N_{\mathrm{text}}+N_{\mathrm{vis}},
$$

缓存长度 $T_{\mathrm{cap}}$ 表示推理过程中 KV cache 需要容纳的最大 token 数，定义为：

$$
T_{\mathrm{cap}}=T_{\mathrm{in}}+N_{\mathrm{out}}.
$$

### 1.3 输入配置

#### 1.3.1 模型配置

用 $\mathcal{M}$ 表示模型配置集合。其中，$b_{\mathrm{LLM}}^{\mathrm{eff}}$、$b_{\mathrm{vis}}^{\mathrm{eff}}$ 与 $b_{\mathrm{proj}}^{\mathrm{eff}}$ 分别表示语言主干、视觉编码器和跨模态投影器的实际有效存储位宽，$b_{\mathrm{KV}}$ 表示 KV cache 元素位宽。对每个模型记录以下信息：

$$
\mathcal{M}=\{P_{\mathrm{LLM}},P_{\mathrm{vis}},P_{\mathrm{proj}},L,H,n_{\mathrm{q}},n_{\mathrm{kv}},d_h,b_{\mathrm{LLM}}^{\mathrm{eff}},b_{\mathrm{vis}}^{\mathrm{eff}},b_{\mathrm{proj}}^{\mathrm{eff}},b_{\mathrm{KV}}\}.
$$

- 参数量覆盖完整 VLM，包括语言主干、视觉编码器、跨模态投影器（projector）、任务头和已加载的适配模块（Adapter）；
- 对混合专家模型（Mixture-of-Experts, MoE），内存按总参数量计算，计算量按活跃参数量计算；
- 权重量化精度与 KV cache 精度分别记录。

#### 1.3.2 板卡配置

用 $\mathcal{H}$ 表示板卡硬件配置集合，其中 $M_{\mathrm{phys}}$ 为物理内存容量，$M_{\mathrm{usable}}$ 为可分配给推理进程的有效内存，$C_{\mathrm{peak}}$ 为理论峰值计算能力，$B_{\mathrm{mem}}$ 为理论内存带宽，$P_{\mathrm{max}}$ 为板卡最大功耗。对每个板卡记录：

$$
\mathcal{H}=\{M_{\mathrm{phys}},M_{\mathrm{usable}},C_{\mathrm{peak}},B_{\mathrm{mem}},P_{\mathrm{max}}\}.
$$

其中，$M_{\mathrm{OS/driver}}$ 为操作系统、驱动和基础运行环境占用的内存，$M_{\mathrm{reserved}}$ 为已被系统或其他计划服务保留、不可分配给推理进程的内存。$M_{\mathrm{usable}}$ 根据实际环境获得：

$$
M_{\mathrm{usable}}=M_{\mathrm{phys}}-M_{\mathrm{OS/driver}}-M_{\mathrm{reserved}}.
$$

对于 Jetson 等统一内存板卡，还应扣除操作系统、CPU 进程和图像预处理等共享内存占用。

## 2. 计算过程
### 2.1 步骤一：内存可部署性计算

#### 2.1.1 静态权重内存

令 $M_{\mathrm{weight}}$ 表示完整 VLM 权重占用的内存，则：

$$
M_{\mathrm{weight}}
=
\frac{
P_{\mathrm{LLM}}b_{\mathrm{LLM}}^{\mathrm{eff}}
+
P_{\mathrm{vis}}b_{\mathrm{vis}}^{\mathrm{eff}}
+
P_{\mathrm{proj}}b_{\mathrm{proj}}^{\mathrm{eff}}
}{8\cdot 2^{30}}
\quad [\mathrm{GiB}].
$$

若还存在任务头、低秩适配（Low-Rank Adaptation, LoRA）或其他模块，应将相应项继续加入分子。

对于模块量化方式一致的情形，$P_{\mathrm{total}}$ 表示完整 VLM 总参数量，$b^{\mathrm{eff}}$ 表示所有模块统一量化时的平均有效位宽，可使用近似式：

$$
M_{\mathrm{weight}}\approx \frac{P_{\mathrm{total}}b^{\mathrm{eff}}}{8\cdot 2^{30}}.
$$

其中分母中的 8 用于将 bit 换算为 Byte；该近似忽略 scale、zero-point、padding 等细粒度存储开销。

#### 2.1.2 KV cache 内存

对于“视觉 token 与文本 token 拼接后进入解码器（decoder）”的 VLM，Transformer 第 $\ell$ 层需要保存 Key 张量 $K^{(\ell)}$ 与 Value 张量 $V^{(\ell)}$。其中，$\ell\in\{1,\ldots,L\}$ 为 Transformer 层索引；$n_{\mathrm{kv},\ell}$ 与 $d_{h,\ell}$ 分别为第 $\ell$ 层的 KV heads 数和单个 attention head 维度：

$$
K^{(\ell)},V^{(\ell)}\in \mathbb{R}^{B\times T_{\mathrm{cap}}\times n_{\mathrm{kv},\ell}\times d_{h,\ell}}.
$$

令 $M_{\mathrm{KV}}$ 表示所有层的 Key–Value（KV）cache 内存。下式中的 $\sum_{\ell=1}^{L}$ 表示从第 1 层到第 $L$ 层累加：

$$
M_{\mathrm{KV}}
=
\frac{
B T_{\mathrm{cap}}
\sum_{\ell=1}^{L}
\left(2n_{\mathrm{kv},\ell}d_{h,\ell}s_{\mathrm{KV}}\right)
}{2^{30}}
\quad [\mathrm{GiB}].
$$

若各层结构相同，则化简为：

$$
M_{\mathrm{KV}}
=
\frac{
2BT_{\mathrm{cap}}Ln_{\mathrm{kv}}d_hs_{\mathrm{KV}}
}{2^{30}}.
$$

定义每 token 的 KV cache 成本：

$$
c_{\mathrm{KV}}
=
\frac{2BLn_{\mathrm{kv}}d_hs_{\mathrm{KV}}}{2^{30}}
\quad [\mathrm{GiB/token}],
$$

则：

$$
M_{\mathrm{KV}}=T_{\mathrm{cap}}c_{\mathrm{KV}}.
$$

注意：若 VLM 通过交叉注意力（cross-attention）将视觉特征作为独立编码器记忆（encoder memory）输入解码器，则视觉 token 不应直接加入解码器自注意力（self-attention）的 $T_{\mathrm{cap}}$。此时，$M_{\mathrm{total}}$ 为总内存，$M_{\mathrm{KV,self}}$ 为 decoder self-attention 的 KV cache 内存，$M_{\mathrm{vision\ memory}}$ 为视觉编码器输出特征及其缓存占用，$M_{\mathrm{KV,cross}}$ 为 cross-attention 中视觉 Key/Value 的缓存内存：

$$
M_{\mathrm{total}}=M_{\mathrm{KV,self}}+M_{\mathrm{vision\ memory}}+M_{\mathrm{KV,cross}}.
$$

#### 2.1.3 辅助内存

令 $M_{\mathrm{aux}}$ 表示辅助内存：

$$
M_{\mathrm{aux}}
=
M_{\mathrm{vision,peak}}
+
M_{\mathrm{workspace}}
+
M_{\mathrm{runtime}}.
$$

其中：

- $M_{\mathrm{vision,peak}}$：图像 tensor、视觉编码器中间特征和 projector 的峰值占用；
- $M_{\mathrm{workspace}}$：attention、通用矩阵乘法（General Matrix Multiplication, GEMM）和量化 kernel 的工作区；
- $M_{\mathrm{runtime}}$：推理框架、驱动、内存池和通信 buffer 等开销。

该项应通过实际内存 profiling 获得；初步估算的示例取值仅用于演示，实际应以实测结果替换。

#### 2.1.4 峰值内存与容量判定

令 $M_{\mathrm{req}}$ 表示任务峰值内存，$\Delta M$ 表示内存裕量：

$$
M_{\mathrm{req}}
=
M_{\mathrm{weight}}
+
M_{\mathrm{KV}}
+
M_{\mathrm{aux}}.
$$

内存裕量定义为：

$$
\Delta M=M_{\mathrm{usable}}-M_{\mathrm{req}}.
$$

内存可部署性二值判定 $\mathrm{Deployable}_{\mathrm{memory}}$ 定义为：

$$
\mathrm{Deployable}_{\mathrm{memory}}
=
\mathbb{I}(\Delta M\ge 0).
$$

其中 $\mathbb{I}(\cdot)$ 为示性函数：括号内条件成立时取 1，否则取 0；$\ge$ 表示大于或等于。

---

### 2.2 步骤二：最大上下文与输出长度

在给定板卡上，$T_{\mathrm{cap,max}}$ 表示由剩余内存约束得到的理论最大缓存长度。$\lfloor\cdot\rfloor$ 为下取整函数：

$$
T_{\mathrm{cap,max}}
=
\left\lfloor
\frac{
M_{\mathrm{usable}}
-M_{\mathrm{weight}}
-M_{\mathrm{aux}}
}{c_{\mathrm{KV}}}
\right\rfloor.
$$

最终可用最大上下文长度 $T_{\max}$ 应取内存上限与模型架构或配置文件允许的最大上下文长度 $T_{\mathrm{model,max}}$ 的较小值；$\min(\cdot,\cdot)$ 表示取较小值：

$$
T_{\max}=\min\left(T_{\mathrm{cap,max}},T_{\mathrm{model,max}}\right).
$$

给定输入后，$N_{\mathrm{out,max}}$ 表示可支持的最大输出 token 数：

$$
N_{\mathrm{out,max}}
=T_{\max}-N_{\mathrm{text}}-N_{\mathrm{vis}}.
$$

### 2.3 步骤三：计算量、时延与能耗筛选

#### 2.3.1 端到端时延分解

令 $T_{\mathrm{e2e}}$ 为端到端推理时延，$T_{\mathrm{preprocess}}$ 为图像与文本预处理时延，$T_{\mathrm{vision}}$ 为视觉编码器前向计算时延，$T_{\mathrm{prefill}}$ 为处理完整输入序列并建立初始 KV cache 的时延，$T_{\mathrm{decode}}$ 为逐 token 自回归生成时延：

$$
T_{\mathrm{e2e}}
=
T_{\mathrm{preprocess}}
+T_{\mathrm{vision}}
+T_{\mathrm{prefill}}
+T_{\mathrm{decode}}.
$$

其中，视觉编码和 prefill 通常更偏计算受限；自回归 decode 通常更易受权重与 KV cache 访问带宽限制。

#### 2.3.2 FLOPs 近似

FLOPs 表示浮点运算次数。对稠密 decoder-only 语言主干，$F_{\mathrm{vision}}$、$F_{\mathrm{prefill}}$、$F_{\mathrm{decode}}$ 和 $F_{\mathrm{task}}$ 分别表示视觉编码、prefill、decode 和单次任务的计算量。$T_{\mathrm{in}}$ 定义为：

$$
T_{\mathrm{in}}=N_{\mathrm{text}}+N_{\mathrm{vis}}.
$$

定义每层 attention 中 query 的总投影维度 $D_Q$：

$$
D_Q=n_{\mathrm{q}}d_h.
$$

prefill 的近似计算量为：

$$
F_{\mathrm{prefill}}
\approx
F_{\mathrm{vision}}
+2P_{\mathrm{LLM}}T_{\mathrm{in}}
+4LD_QT_{\mathrm{in}}^2.
$$

生成 $N_{\mathrm{out}}$ 个 token 的 decode 近似计算量为：

$$
F_{\mathrm{decode}}
\approx
2P_{\mathrm{LLM}}N_{\mathrm{out}}
+4LD_Q\left[
N_{\mathrm{out}}T_{\mathrm{in}}
+\frac{N_{\mathrm{out}}(N_{\mathrm{out}}-1)}{2}
\right].
$$

当 $D_Q=H$ 时，上式退化为常用的 $4LHT^2$ 形式，其中一般序列长度 $T$ 表示 attention 的输入 token 数。对于 Qwen3-VL-4B，$D_Q=4096\neq H=2560$，因此应使用 $D_Q$ 而非 $H$ 计算 attention 项。

总计算量：

$$
F_{\mathrm{task}}=F_{\mathrm{prefill}}+F_{\mathrm{decode}}.
$$

对于混合专家模型（MoE），以上公式中的 $P_{\mathrm{LLM}}$ 应替换为活跃参数量 $P_{\mathrm{active}}$，即一次前向推理中实际被路由并参与计算的参数量；$M_{\mathrm{weight}}$ 仍按总参数量计算。

#### 2.3.3 Roofline 下界与任务判定

令 $T_{\mathrm{infer}}$ 为理论推理时延下界，$F_{\mathrm{task}}$ 为任务总计算量，$\eta_C$ 为实际计算利用率，$C_{\mathrm{peak}}$ 为理论峰值计算能力，$D_{\mathrm{mem}}$ 为任务执行过程中的总访存量，$\eta_B$ 为实际内存带宽利用率，$B_{\mathrm{mem}}$ 为理论内存带宽：

$$
T_{\mathrm{infer}}
\gtrsim
\max\left(
\frac{F_{\mathrm{task}}}{\eta_C C_{\mathrm{peak}}},
\frac{D_{\mathrm{mem}}}{\eta_B B_{\mathrm{mem}}}
\right),
$$

实际部署应以 profile 得到的 $T_{\mathrm{e2e}}$、TTFT（time to first token，首个 token 生成时延）和 decode throughput（解码吞吐率，token/s，即每秒生成 token 数）为准。

令 $E_{\mathrm{task}}$ 为单次任务能耗，$P_{\mathrm{avg}}$ 为任务执行期间的平均功耗，则任务能耗可由实测平均功耗估计：

$$
E_{\mathrm{task}}=P_{\mathrm{avg}}T_{\mathrm{e2e}}.
$$

最终“任务可部署”要求以下条件同时成立：$T_{\mathrm{deadline}}$ 为任务允许的最大端到端时延，$E_{\mathrm{budget}}$ 为单次任务的能量预算，$P_{\mathrm{peak}}$ 为任务运行期间的峰值功耗，$P_{\mathrm{thermal}}$ 为热设计或热控允许的最大功耗：

$$
\begin{cases}
M_{\mathrm{req}}\le M_{\mathrm{usable}},\\
T_{\mathrm{e2e}}\le T_{\mathrm{deadline}},\\
E_{\mathrm{task}}\le E_{\mathrm{budget}},\\
P_{\mathrm{peak}}\le P_{\mathrm{thermal}}.
\end{cases}
$$

## 3. Qwen3-VL-4B 示例

本节将前述通用流程具体化为 Qwen3-VL-4B 的单板推理示例。模型结构参数采用其 `config.json` 对应的文本解码器配置；参数量按语言主干约 $3.7$B、视觉编码器与 projector 合计约 $0.74$B 计。输入长度、量化配置与辅助内存均为本节固定的输入与运行假设，实际部署时应以 processor 输出和板端 profile 替换。

### 3.1 固定配置

| 项目 | 取值 |
|---|---:|
| 语言主干参数量 $P_{\mathrm{LLM}}$ | $3.70$B |
| 视觉编码器与 projector 参数量 $P_{\mathrm{vis}}+P_{\mathrm{proj}}$ | $0.74$B |
| 完整 VLM 总参数量 $P_{\mathrm{total}}$ | 约 $4.44$B |
| Transformer 层数 $L$ | $36$ |
| hidden size $H$ | $2560$ |
| query heads $n_{\mathrm{q}}$ | $32$ |
| KV heads $n_{\mathrm{kv}}$ | $8$ |
| head dimension $d_h$ | $128$ |
| attention query 投影维度 $D_Q=n_{\mathrm{q}}d_h$ | $4096$ |
| batch size $B$ | $1$ |
| 文本输入 $N_{\mathrm{text}}$ | $1024$ |
| 视觉 token $N_{\mathrm{vis}}$ | $1024$ |
| 最大输出长度 $N_{\mathrm{out}}$ | $1024$ |
| KV cache 精度 | BF16，$s_{\mathrm{KV}}=2$ Byte |
| 视觉编码计算量 $F_{\mathrm{vision}}$ | $0.05$ TFLOPs（示例假设） |
| 辅助内存 $M_{\mathrm{aux}}$ | $1.60$ GiB（示例假设） |

因此：

$$
T_{\mathrm{in}}=N_{\mathrm{text}}+N_{\mathrm{vis}}=1024+1024=2048,
$$

$$
T_{\mathrm{cap}}=T_{\mathrm{in}}+N_{\mathrm{out}}=2048+1024=3072.
$$

### 3.2 W4 权重内存

语言主干采用 W4，取有效平均位宽 $b_{\mathrm{LLM}}^{\mathrm{eff}}=4.25$ bit；视觉编码器与 projector 保持 BF16。

语言主干权重内存记为 $M_{\mathrm{LLM}}$：

$$
M_{\mathrm{LLM}}
=
\frac{3.70\times10^9\times4.25}{8\times2^{30}}
=1.831\ \mathrm{GiB}.
$$

视觉编码器与投影器权重内存之和记为 $M_{\mathrm{vis+proj}}$：

$$
M_{\mathrm{vis+proj}}
=
\frac{0.74\times10^9\times16}{8\times2^{30}}
=1.378\ \mathrm{GiB}.
$$

故：

$$
M_{\mathrm{weight}}
=1.831+1.378
=3.209\ \mathrm{GiB}.
$$

### 3.3 KV cache 内存

每 token 的 KV cache 成本为：

$$
c_{\mathrm{KV}}
=
\frac{2\times1\times36\times8\times128\times2}{2^{30}}
=1.373\times10^{-4}\ \mathrm{GiB/token}.
$$

即：

$$
c_{\mathrm{KV}}=144\ \mathrm{KiB/token}.
$$

因此：

$$
M_{\mathrm{KV}}
=3072\times144\ \mathrm{KiB}
=0.422\ \mathrm{GiB}.
$$

### 3.4 峰值内存需求

本例固定：

$$
M_{\mathrm{aux}}
=M_{\mathrm{vision,peak}}+M_{\mathrm{workspace}}+M_{\mathrm{runtime}}
=0.5+0.6+0.5
=1.6\ \mathrm{GiB}.
$$

因此：

$$
M_{\mathrm{req}}
=M_{\mathrm{weight}}+M_{\mathrm{KV}}+M_{\mathrm{aux}}
=3.209+0.422+1.600
=5.231\ \mathrm{GiB}.
$$

即，Qwen3-VL-4B 在该 W4、单图、$1024$ 视觉 token、$1024$ 输出 token 条件下的估算峰值内存为：

$$
\boxed{M_{\mathrm{req}}=5.231\ \mathrm{GiB}}.
$$

其中 $M_{\mathrm{aux}}$ 为初始示例估计，最终数值应以具体推理框架、量化后端和板端 profile 的峰值显存替换。

### 3.5 不同权重量化配置比较

保持相同输入长度、KV cache 精度与辅助内存假设：

| 语言主干精度 | $M_{\mathrm{weight}}$ | $M_{\mathrm{KV}}$ | $M_{\mathrm{aux}}$ | $M_{\mathrm{req}}$ |
|---|---:|---:|---:|---:|
| BF16 | 8.270 GiB | 0.422 GiB | 1.600 GiB | 10.292 GiB |
| W8，有效 $8.5$ bit | 5.039 GiB | 0.422 GiB | 1.600 GiB | 7.061 GiB |
| W4，有效 $4.25$ bit | 3.209 GiB | 0.422 GiB | 1.600 GiB | 5.231 GiB |

### 3.6 板卡容量判定示例

以可用内存 $6.8$、$13.6$ 和 $27.2$ GiB 的三类示例板卡为例：

| 示例板卡 | $M_{\mathrm{usable}}$ | BF16 | W8 | W4 |
|---|---:|---|---|---|

$$

该组合通过内存门槛。对 W8 权重量化配置，将其内存裕量记为 $\Delta M_{\mathrm{W8}}$：

$$
\Delta M_{
\mathrm{W8}}
=6.8-7.061
=-0.261\ \mathrm{GiB},
$$

因此应记为边界不可部署；实际运行中容易受框架版本、量化实现和输入波动影响而触发 OOM（out-of-memory，内存不足错误）。

### 3.7 计算量近似

由于该模型的 attention query 投影维度为：

$$
D_Q=n_{\mathrm{q}}d_h=32\times128=4096,
$$

prefill 计算量为：

$$
\begin{aligned}
F_{\mathrm{prefill}}
&\approx F_{\mathrm{vision}}
+2P_{\mathrm{LLM}}T_{\mathrm{in}}
+4LD_QT_{\mathrm{in}}^2\\
&=0.05+15.155+2.474\\
&=17.679\ \mathrm{TFLOPs}.
\end{aligned}
$$

生成 $1024$ 个 token 的 decode 计算量为：

$$
\begin{aligned}
F_{\mathrm{decode}}
&\approx2P_{\mathrm{LLM}}N_{\mathrm{out}}
+4LD_Q\left[N_{\mathrm{out}}T_{\mathrm{in}}
+\frac{N_{\mathrm{out}}(N_{\mathrm{out}}-1)}{2}\right]\\
&=7.578+1.546\\
&=9.123\ \mathrm{TFLOPs}.
\end{aligned}
$$

因此：

$$
F_{\mathrm{task}}
=F_{\mathrm{prefill}}+F_{\mathrm{decode}}
=26.803\ \mathrm{TFLOPs}.
$$

该 FLOPs 结果仅用于比较板卡上的理论计算压力。实际部署结论仍应由板端 profile 的 $T_{\mathrm{e2e}}$、TTFT、decode throughput、功耗和峰值内存共同给出。
